# 科技新闻自动收集项目 PRD

## 1. 项目背景

本项目用于快速验证一个自动化新闻收集流程：

> 每天自动收集科技新闻，通过 AI 生成中文摘要，再把结果沉淀到 Notion。

当前阶段优先目标不是做完整商业化产品，而是先做出一个可以运行、可以展示、可以继续迭代的第一版自动化工作流。

## 2. 项目目标

### 2.1 第一阶段目标

- 使用本地 n8n 搭建新闻自动收集工作流。
- 从 RSS 新闻源读取科技新闻。
- 调用 DeepSeek API 生成中文摘要。
- 将新闻标题、链接、摘要、状态和日期写入 Notion 数据库。
- 先支持手动触发测试，跑通后再改为每天早上 8 点自动执行。

### 2.2 长期目标

- 支持多个科技新闻源。
- 支持去重、筛选、重要性判断。
- 支持“单条摘要 -> 每日科技日报”两段式文本生成。
- 支持 Notion 内容库长期沉淀。
- 后续根据验证结果，逐步考虑商业化、订阅分发或多平台发布。

## 3. 第一阶段范围

### 3.1 包含功能

- 手动触发工作流。
- 读取一个 RSS 源。
- 限制只处理 1 条新闻，用于最小测试。
- 调用 DeepSeek 生成 3 句话以内中文摘要。
- 写入 Notion 数据库。

### 3.2 暂不包含

- 多 RSS 源聚合。
- 新闻去重。
- 新闻正文深度抓取。
- 每日整篇日报生成。
- 自动定时触发。
- 错误通知。
- 成本统计。
- 用户系统、付费系统、商业化后台。

## 4. 技术方案

### 4.1 技术栈

- 自动化平台：n8n 本地版
- 新闻来源：RSS
- AI 生成：DeepSeek API
- 内容沉淀：Notion 数据库
- 当前工作流文件：
  - `tech-news-deepseek-notion.workflow.json`

### 4.2 当前工作流

```text
Manual Trigger
-> RSS Read
-> Limit 1 条
-> HTTP Request 调用 DeepSeek
-> HTTP Request 写入 Notion
```

### 4.3 第一版 RSS 源

```text
https://techcrunch.com/feed/
```

后续可扩展为多个科技新闻源。

## 5. Notion 数据库设计

第一阶段 Notion 数据库字段：

| 字段名 | 类型 | 说明 |
| --- | --- | --- |
| 标题 | Title | 新闻标题 |
| 链接 | URL | 原文链接 |
| 摘要 | Rich text / Text | DeepSeek 生成的中文摘要 |
| 状态 | Select | 第一阶段默认写入“已收集” |
| 日期 | Date | 写入 Notion 的时间 |

## 6. 配置项

导入 n8n 工作流后，需要替换以下占位符：

```text
你的_DEEPSEEK_API_KEY
你的_NOTION_API_KEY
你的_NOTION_DATABASE_ID
```

建议不要把真实 API Key 写入 Git 或公开文档。当前最小测试可以先在 n8n 节点里手动填写，后续再改为 n8n Credentials 或环境变量。

## 7. 验收标准

第一阶段完成标准：

- n8n 工作流可以成功导入。
- 手动执行工作流不会报错。
- RSS 节点能读取 TechCrunch 新闻。
- DeepSeek 节点能返回中文摘要。
- Notion 数据库能新增 1 条新闻记录。
- 新增记录至少包含标题、链接、摘要、状态和日期。

## 8. 下一阶段计划

第一阶段跑通后，再按下面顺序迭代：

1. 将 `Manual Trigger` 改为每天早上 8 点定时触发。
2. 将 `Limit 1 条` 改为处理 5-10 条新闻。
3. 增加多个 RSS 源。
4. 增加基础去重逻辑。
5. 增加第二次 AI 生成，把多条摘要合成每日科技日报。
6. 增加错误记录和失败提醒。
7. 将 API Key 改为 n8n Credentials 或环境变量管理。
